- **출처 웹사이트**: DONG BANG FOOD (food.jpg 분석)
- **분석 날짜**: 2026-06-22
- **적용 프로젝트**: 포트폴리오 웹사이트

## CSS 변수 정의

```css
:root {
  /* Primary Colors */
  --color-primary: #E8540C;
  --color-primary-light: #F2784A;
  --color-primary-dark: #B83E08;

  /* Secondary Colors */
  --color-secondary: #F5E6D3;
  --color-accent: #2BB89C;

  /* Background Colors */
  --color-bg-primary: #F5E6D3;
  --color-bg-secondary: #FFFFFF;

  /* Text Colors */
  --color-text-primary: #2B1A0F;
  --color-text-secondary: #5C4A3A;
  --color-text-muted: #8C7A6A;

  /* Interactive Colors */
  --color-button-primary: #E8540C;
  --color-button-hover: #C8460A;
  --color-link: #2BB89C;
  --color-link-hover: #1F9580;
}
```

## 컬러 사용 가이드

### Primary (`--color-primary`, `#E8540C`)
- 브랜드 정체성을 가장 강하게 드러내는 색
- 히어로 섹션 배경, 메인 타이틀, CTA 버튼 등 시선을 가장 먼저 끌어야 하는 요소에 사용
- 과도하게 넓은 면적보다는 핵심 포인트 위주로 사용 권장

### Primary Light / Dark
- `--color-primary-light` (`#F2784A`): 호버 상태가 아닌 보조 강조, 일러스트/아이콘 컬러
- `--color-primary-dark` (`#B83E08`): 그림자, 텍스트 위 배경 대비가 필요한 강조 영역

### Secondary (`--color-secondary`, `#F5E6D3`)
- 전체 페이지의 기본 톤을 잡아주는 차분한 베이스 컬러
- 섹션 배경, 카드 배경 등 넓은 면적에 사용해 Primary의 강렬함을 완충

### Accent (`--color-accent`, `#2BB89C`)
- Primary(오렌지)의 보색으로 신선함과 생동감을 더하는 포인트 컬러
- 배지, 보조 카드, 링크 등 "다름"을 표현해야 하는 요소에 제한적으로 사용

### Background
- `--color-bg-primary` (`#F5E6D3`): 페이지 기본 배경
- `--color-bg-secondary` (`#FFFFFF`): 콘텐츠 카드, 가독성이 중요한 콘텐츠 박스 배경

### Text
- `--color-text-primary` (`#2B1A0F`): 본문 제목 등 핵심 텍스트
- `--color-text-secondary` (`#5C4A3A`): 본문 설명, 부가 텍스트
- `--color-text-muted` (`#8C7A6A`): 캡션, 타임스탬프 등 우선도가 낮은 텍스트

### Interactive
- `--color-button-primary` (`#E8540C`) → hover 시 `--color-button-hover` (`#C8460A`)로 어둡게 전환해 클릭 가능 상태 명확히 표현
- `--color-link` (`#2BB89C`) → hover 시 `--color-link-hover` (`#1F9580`)로 더 진한 톤으로 전환

## 접근성 체크 포인트
- `--color-text-primary` on `--color-bg-primary`: 명도 대비 충분 (다크 브라운 on 라이트 크림)
- `--color-bg-secondary`(화이트) 위에는 `--color-primary` 텍스트 사용 시 대비 확인 필요 — 작은 텍스트에는 `--color-primary-dark` 권장
- 버튼 텍스트는 항상 화이트(`#FFFFFF`) 고정 사용

## 반응형 고려사항

### 다크 모드 적용
다크 모드에서는 명도를 반전시키는 대신, 각 역할(배경/텍스트/강조)의 톤을 유지하면서 어두운 버전으로 치환합니다.

```css
[data-theme='dark'] {
  --color-bg-primary: #1A1410;
  --color-bg-secondary: #241C16;
  --color-text-primary: #F5E6D3;
  --color-text-secondary: #C9B6A4;
  --color-text-muted: #8C7A6A;
  --color-primary: #FF6B2E;       /* 어두운 배경에서 채도/명도를 살짝 올려 대비 확보 */
  --color-primary-light: #FF8F5C;
  --color-primary-dark: #D9540F;
  --color-secondary: #2E2620;
  --color-accent: #38D6B6;        /* 어두운 배경에서 더 밝게 보정 */
}
```
- Primary/Accent 컬러는 다크 배경에서 묻히지 않도록 명도를 5~10% 정도 올려서 사용
- 텍스트 컬러는 라이트 모드의 배경색(`#F5E6D3` 계열)을 그대로 텍스트로 재사용해 톤의 연속성 유지

### 화면 크기별 적용 방법
- **모바일 (~767px)**: 컬러 영역을 단순화 — Primary는 버튼/타이틀에만, Secondary 배경 단일 톤 유지로 시각적 피로도 감소
- **태블릿 (768px~1023px)**: Background Secondary(카드)와 Background Primary(페이지) 구분을 명확히 해 레이아웃 위계 표현
- **데스크톱 (1024px~)**: Accent 컬러를 호버/인터랙션 디테일에까지 확장 적용 가능 (예: 카드 호버 시 보더 컬러를 Accent로 전환)

### 다크모드 전환 시 주의사항
- 단순 반전(invert)은 금지 — 브랜드 컬러(Primary, Accent)의 색상값 자체는 유지하고 명도만 조정
- 다크모드에서도 버튼 텍스트는 항상 화이트(`#FFFFFF`) 고정 사용
